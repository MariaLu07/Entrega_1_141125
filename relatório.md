A arquitetura do projeto será dividida entre front-end e back-end, e cada parte terá um papel importante dentro do sistema. No front-end, serão usados HTML, CSS e JavaScript. O HTML será responsável por montar a estrutura das páginas, o CSS deixará tudo organizado e visualmente bonito, e o JavaScript cuidará da parte interativa, fazendo a tela responder às ações do usuário.

O back-end será feito em Python, que ficará responsável por todo o processamento interno. É ele que receberá as informações enviadas pelo front-end, verificará se está tudo correto, aplicará as regras de funcionamento do sistema e organizará a forma como os dados serão guardados. Basicamente, ele fará o “trabalho por trás das telas”, garantindo que tudo funcione como deve.

Os dados serão coletados principalmente através de formulários no próprio site. Quando o usuário preencher essas informações, o front-end fará uma verificação inicial para garantir que está tudo no formato certo. Depois disso, esses dados serão enviados ao back-end, onde passarão por outra verificação mais completa antes de serem armazenados no banco de dados.

Durante o funcionamento do sistema, o front-end e o back-end estarão sempre se comunicando. Assim, o usuário receberá respostas claras, como mensagens de confirmação, avisos de erro ou atualizações na tela. Além disso, para manter tudo organizado durante o desenvolvimento, cada alteração feita no código deverá ser registrada com commits, garantindo controle de versão e organização do projeto.

Essa arquitetura mostra de forma clara como o sistema será construído e como as informações circularão entre as partes que compõem a plataforma.
